Codex CLI needs both a custom provider in `~/.codex/config.toml` and an API key in `~/.codex/auth.json`.

## Install Codex CLI

```bash
npm install -g @openai/codex
```

## `~/.codex/config.toml`

```toml
personality = "pragmatic"
model = "gpt-5.4"
model_provider = "appleai"

[model_providers.appleai]
name = "appleai"
base_url = "http://160.187.0.165:8138/v1"
wire_api = "responses"
```

## `~/.codex/auth.json`

```json
{
  "OPENAI_API_KEY": "YOUR_APPLEAI_API_KEY"
}
```

## What to verify

1. `model_provider` matches the provider block name exactly as `appleai`
2. `base_url` points to `http://160.187.0.165:8138/v1`
3. `OPENAI_API_KEY` in `~/.codex/auth.json` contains the correct API key
4. Use normal ASCII quotes in TOML, not smart quotes

## Launch

```bash
codex
```

If Codex CLI starts but fails to answer, recheck both the provider settings in `~/.codex/config.toml` and the API key in `~/.codex/auth.json`.
