Codex CLI dùng một provider block trong `~/.codex/config.toml` và một API key trong `~/.codex/auth.json`.

## Cài đặt Codex CLI

```bash
npm install -g @openai/codex
```

## `~/.codex/config.toml`

```toml
model = "openai/gpt-5"
model_provider = "AppleAI"
model_reasoning_effort = "high"
sandbox_mode = "workspace-write"

[model_providers.AppleAI]
name = "AppleAI"
base_url = "https://api.appleai.codes/v1"
wire_api = "responses"

[agents.subagent]
model = "openai/gpt-5-mini"
```

## `~/.codex/auth.json`

```json
{
  "OPENAI_API_KEY": "YOUR_APPLEAI_API_KEY",
  "auth_mode": "apikey"
}
```

## Những gì cần xác minh

1. `model_provider` phải khớp chính xác với tên provider block
2. `base_url` phải bao gồm `/v1`
3. Model route đã chọn phải tồn tại trong tài khoản của bạn

## Chạy thử

```bash
codex
```

Nếu Codex CLI thấy provider nhưng không trả lời được, hãy kiểm tra model route đó bằng lệnh `curl` trong [Cấu hình dùng chung](/vi/getting-started/shared-configuration).
