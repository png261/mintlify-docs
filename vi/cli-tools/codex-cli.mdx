Codex CLI cần cả provider tùy chỉnh trong file `config.toml` và API key trong file `auth.json`.

## Cài đặt Codex CLI

### macOS và Linux

```bash
npm install -g @openai/codex
```

Dùng:

- `~/.codex/config.toml`
- `~/.codex/auth.json`

### Windows PowerShell

```powershell
npm install -g @openai/codex
```

Dùng:

- `%USERPROFILE%\.codex\config.toml`
- `%USERPROFILE%\.codex\auth.json`

## `config.toml`

```toml
personality = "pragmatic"
model = "gpt-5.4"
model_provider = "appleai"

[model_providers.appleai]
name = "appleai"
base_url = "http://160.187.0.165:8138/v1"
wire_api = "responses"
```

## `auth.json`

```json
{
  "OPENAI_API_KEY": "YOUR_APPLEAI_API_KEY"
}
```

## Những gì cần xác minh

1. `model_provider` phải khớp chính xác với tên provider block là `appleai`
2. `base_url` phải trỏ tới `http://160.187.0.165:8138/v1`
3. `OPENAI_API_KEY` trong file `auth.json` của bạn phải chứa đúng API key
4. Hãy dùng dấu ngoặc kép ASCII bình thường trong TOML, không dùng smart quotes

## Chạy thử

```bash
codex
```

Nếu Codex CLI khởi động nhưng không trả lời được, hãy kiểm tra lại cả cấu hình provider và API key trong đúng file của hệ điều hành bạn đang dùng.
