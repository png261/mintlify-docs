Hãy dùng trang này khi bạn muốn có cách thiết lập nhanh nhất.

## Chọn hệ điều hành của bạn

- macOS: Dùng Terminal hoặc iTerm và lưu file cấu hình dưới `~/.tool-name/`
- Linux: Dùng shell quen thuộc của bạn và cùng quy ước `~/.tool-name/`
- Windows: Dùng PowerShell và lưu file cấu hình dưới `%USERPROFILE%\.tool-name\`
- WSL: Hãy làm theo ví dụ Linux ở bên trong môi trường WSL

## Dùng lại các giá trị này ở mọi nơi

### macOS và Linux

```bash
export APPLEAI_BASE_URL=https://api.appleai.codes/v1
export APPLEAI_API_KEY=YOUR_APPLEAI_API_KEY
export APPLEAI_MODEL=anthropic/claude-sonnet-4
```

### Windows PowerShell

```powershell
$env:APPLEAI_BASE_URL="https://api.appleai.codes/v1"
$env:APPLEAI_API_KEY="YOUR_APPLEAI_API_KEY"
$env:APPLEAI_MODEL="anthropic/claude-sonnet-4"
```

## Luồng cơ bản

1. Tạo hoặc sao chép một API key của `appleai.codes`.
2. Chọn một model path đã được route như `anthropic/claude-sonnet-4` hoặc `openai/gpt-5`.
3. Mở hướng dẫn cho công cụ bạn dùng và dán các giá trị dùng chung vào phần cấu hình.
4. Chạy một request đơn giản để xác nhận endpoint và key hoạt động cùng nhau.

## Kiểm tra kết nối trước

### macOS và Linux

```bash
curl https://api.appleai.codes/v1/models \
  -H "Authorization: Bearer $APPLEAI_API_KEY"
```

### Windows PowerShell

```powershell
curl.exe https://api.appleai.codes/v1/models `
  -H "Authorization: Bearer $env:APPLEAI_API_KEY"
```

Nếu lệnh này chạy thành công, key của bạn hợp lệ và base URL có thể truy cập được trong shell hiện tại của bạn.

## Thứ tự nên làm

1. Bắt đầu với **Claude Code**, **Codex CLI** hoặc **OpenCode**. Các công cụ này có file cấu hình trực tiếp.
2. Chuyển sang **Cursor**, **Cline**, **Continue** hoặc **Roo** khi bạn muốn tích hợp trong editor.
3. Chỉ dùng luồng **Công cụ chỉ hỗ trợ MITM** khi sản phẩm không cho bạn nhập custom base URL.
