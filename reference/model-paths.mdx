Most tools in this docs set expect a routed model path in the format `provider/model-id`.

## Common examples

| Provider | Example route |
| --- | --- |
| Anthropic | `anthropic/claude-sonnet-4` |
| Anthropic | `anthropic/claude-opus-4.1` |
| OpenAI | `openai/gpt-5` |
| Google | `google/gemini-2.5-pro` |
| xAI | `xai/grok-4` |

## Keep route names stable

When you compare tools, keep the routed model path the same. That helps you separate model behavior from client behavior.

## Tool-specific differences

- Most tools use the raw route, such as `anthropic/claude-sonnet-4`
- Some provider wrappers prepend a local alias, such as `appleai/claude-sonnet-4`
- Some tools cache model lists and need a restart after you change routes

Use the exact names your account exposes. If a route fails, validate it with the `curl` check from [Shared configuration](/getting-started/shared-configuration).
